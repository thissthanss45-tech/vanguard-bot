# Руководство пользователя Vanguard Bot

## 1) Что делает бот
Vanguard Bot помогает быстро оценить тикер по трем источникам:
- Рыночные данные (цена, тренд, волатильность, объем)
- Rule-based модель (вероятности роста/снижения)
- AI-разбор (сценарии, уровни, риск-менеджмент)
- Межрыночные корреляции (SPY и BTC)
- Бэктест базовой RSI-стратегии

Отдельно бот формирует:
- Лаг данных рынка
- Лаг новостей
- Прогноз на ближайшие 3 дня (Д1/Д2/Д3)
- Блок Действия трейдера (вне позиции / в позиции)
- График цены с SMA20/SMA50
- Алерты (например, RSI ниже порога)

---

## 2) Быстрый старт
1. Открой бота в Telegram.
2. Нажми /start.
3. Выбери:
   - 📈 AI-Анализ — полный отчет по тикеру
   - 📚 Тикеры — быстрый выбор тикера по категориям
   - 🗞 Новости — обзор новостей и AI-резюме
   - 🧠 Модели — выбор AI-модели
   - ⚙️ Настройки — provider и риск-профиль

Если бот отвечает не так, как ожидается:
- Нажми /reset
- Затем снова /start

---

## 3) Главное меню: что означает каждая кнопка

### 📈 AI-Анализ
Полный анализ тикера:
- Метаданные инструмента (что это за тикер)
- Лаг данных и время последней свечи
- Технические индикаторы (SMA20/50, EMA200, RSI, ATR, ADX)
- Вероятности роста/снижения
- Прогноз 3 дня (Д1, Д2, Д3)
- AI-разбор
- Блок Действия трейдера

### 🎯 Сигнал по этому тикеру
Короткий итоговый сигнал по последнему проанализированному тикеру:
- Базовый bias
- Лаг рынка и лаг новостей
- Сводный вывод (вход/ждать/сокращать риск)
- Краткий план действий

### 📚 Тикеры
Категории инструментов:
- 🟣 Крипта
- 🔵 Акции
- 🟡 Сырьё
- 🟢 Форекс

Внутри категории:
- Кнопки тикеров
- ⬅️ Предыдущие / ➡️ Следующие (постранично)
- ↩️ Назад (выход)
- 🏠 В главное меню (быстрый выход)

### 🗞 Новости
Новости по тикеру + AI-сводка:
- Сырые заголовки
- Лаг новостей (насколько свежая последняя новость)
- AI-резюме тональности и контекста

### 🧠 Модели
Выбор AI-модели:
- 🦙 Llama
- 🐋 DeepSeek

### ⚙️ Настройки
- provider: какой AI-провайдер используется
- risk: профиль риска в формулировках рекомендаций
  - conservative
  - balanced
  - aggressive

### 🆘 Помощь
Краткая справка по командам и логике бота.

### 📊 /stats
Сводка использования: сколько раз запускались анализ, новости, сигналы и команды.

### 🌐 /lang ru|en
Переключение языка интерфейса (RU/EN) для системных сообщений.

### 🔔 /alert
Управление алертами RSI:
- `/alert list`
- `/alert add BTC-USD 30`
- `/alert del BTC-USD`

## 4) Как правильно анализировать тикер
Рекомендуемый сценарий:
1. Нажми 📈 AI-Анализ.
2. Введи тикер вручную или выбери через 📚 Тикеры.
3. Прочитай полный отчет.
4. При необходимости нажми 🎯 Сигнал по этому тикеру для краткого решения.

---

## 5) Как читать отчет

### Дата анализа
Время формирования отчета (UTC).

### Что это
Расшифровка тикера: название инструмента, тип, биржа, валюта.

### Лаг данных
Насколько свежий рыночный срез.
Пример: Лаг данных 9 ч 30 мин = последняя свеча была 9.5 часов назад.

### Rule-based: bias и вероятности
- Bias показывает направление (бычий/медвежий/нейтральный)
- Вероятности показывают перевес одной стороны

### Прогноз 3 дня
- Д1: прогноз на ближайший день
- Д2: следующий день
- Д3: третий день

Это краткосрочный сценарный блок, а не гарантия движения.

### Фильтр качества
Если рынок некачественный (например, слабый тренд), бот может дать:
- Ждать (фильтр качества)

### Действия трейдера
Всегда 2 строки:
- Что делать вне позиции
- Что делать, если позиция уже открыта

---

## 6) Настройки: простое объяснение
- provider=deepseek или groq — какой AI пишет текстовый разбор
- risk=balanced/conservative/aggressive — стиль риск-рекомендаций

Важно: базовые вероятности считает rule-based модель, а AI в основном объясняет и структурирует вывод.

---

## 7) Команды
- /start — открыть главное меню
- /help — показать справку
- /reset — сбросить текущий режим
- /stats — статистика использования
- /lang ru|en — переключение языка
- /alert list|add|del — алерты по RSI
- /cancel — вернуться в меню
- /srart — резервный алиас (если опечатался)

---

## 8) Если бот завис или не отвечает
1. В Telegram отправь /reset.
2. Затем отправь /start.
3. Повтори действие.

Если проблема повторяется:
- Закрой долгую цепочку сообщений
- Начни новый сценарий через /start
- Используй тикер из списка 📚 Тикеры

---

## 9) Production-режим
Поддерживаются 2 режима запуска:
- Polling (по умолчанию)
- Webhook (`USE_WEBHOOK=true` + `WEBHOOK_URL`)

Для продакшена рекомендуется:
- Включить whitelist Telegram ID (`AUTH_WHITELIST=12345,67890`)
- Использовать `CACHE_BACKEND=redis` или `diskcache`
- Настроить `SENTRY_DSN` для трекинга ошибок
- Запускать стек через `docker-compose.prod.yml` (bot + redis + healthcheck)
- Перед деплоем выполнять `bash scripts/smoke_test.sh`

---

## 10) Ограничения
- Бот не является финансовым советником.
- Сигналы не гарантируют результат.
- Лаг данных и новостей зависит от источников.

Используй риск-менеджмент и не принимай решение только по одному сигналу.
